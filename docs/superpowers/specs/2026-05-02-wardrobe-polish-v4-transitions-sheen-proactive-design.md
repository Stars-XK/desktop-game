# 下一步设计：衣橱整屏切换节奏 · SSR 边框高光进化 · 主动陪伴更聪明

**Goal:** 继续提升“暖暖感”的节奏与高级感：衣橱在分类/筛选切换时有整屏过渡，SSR 边框高光更精致，同时主动陪伴能“看懂你在搭配”，减少打扰并在你停下来时给贴心建议。

**约束**
- 不引入侵权素材；全部程序化/自制实现。
- 不切 URP/HDRP；仍使用 Built-in + UGUI。

---

## 1) 衣橱整屏切换节奏（Screen Transition）

### 1.1 触发点
- 切换分类、切换标签筛选、搜索条件变化、收藏/拥有过滤变化时，统一走“淡出旧卡片 → 生成新卡片 → 入场”。

### 1.2 动画形式（克制、不卡）
- 旧卡片：快速淡出（0.10~0.14s）+ 轻下移（2px）
- 新卡片：已存在的 stagger 入场（0.94→1 + alpha 0→1 + 上浮 4px）

---

## 2) SSR 边框高光进化（Frame Sheen v2）

### 2.1 高光层级
- FrameHighlight：静态淡高光（已存在，按稀有度调透明度）
- FrameSheen：一条非常淡的边框扫光（SSR 启用，周期较长，存在感“贵”）

### 2.2 目标
- SSR 一眼可见但不刺眼；扫光频率低、强度轻，偏“质感”而不是“灯带”。

---

## 3) 主动陪伴更聪明（Proactive Companion v2）

### 3.1 行为准则
- 你在频繁试穿/点卡片/切分类时：降低主动插话（不打断）
- 你停下来（例如 6s 无衣橱操作）且到达最小间隔：主动给一条“搭配建议/夸夸/颜色推荐”

### 3.2 技术实现（最小可用）
- 存档新增 `lastWardrobeActionUnix`（每次衣橱关键操作更新）
- `ProactiveCompanion` 基于 `lastWardrobeActionUnix` 判断“正在搭配/已停下”

---

## 4) 里程碑记忆更“可用”
- 在 system prompt 中加入一条明确规则：当里程碑里出现“喜欢摸头/戳脸/撒娇语气”等时，优先按该偏好组织语气与互动策略（减少随机性）。

---

## 5) 验收标准
- 切换分类/筛选时：旧卡片淡出，新卡片入场（有节奏，不硬切）
- SSR：边框有“低频淡扫光”的贵气效果
- 主动陪伴：搭配频繁时不打扰；停下来后再温柔插一句建议
