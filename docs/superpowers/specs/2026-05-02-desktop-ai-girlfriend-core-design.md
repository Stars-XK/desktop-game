# Desktop AI 女友 Core（人设·记忆·主动陪伴·情绪语音）— Design Spec

**Goal:** 在现有“桌面宠物 + 衣橱展示/抽屉 + 文本聊天 + TTS + STT”的基础上，把体验升级为“更像闪耀暖暖试衣间氛围的桌面 AI 女友”：更稳定的人设、更长的记忆、更主动的陪伴、更贴合情绪的语音表达，并保持“一键 Play 可用”。

**重要说明（避免侵权与安全边界）**
- 目标是做“高级感、粉紫金、玻璃拟态、试衣间氛围”的原创 UI 语言与体验节奏，不复制/不使用闪耀暖暖的任何原始素材、字体、UI 贴图或角色资源。
- 这是陪伴型应用：默认加入“不过度依赖/不替代现实关系/不提供专业治疗建议”的系统边界提示（不会破坏沉浸，但要可控、可关）。

**现状能力（已具备）**
- 文本对话：`UIManager` → `AIManager` → `OpenAILLMProvider`
- 语音输出：`AIManager` → `AzureTTSProvider` → `AudioSource`，并有基础口型
- 语音输入：`VoiceInputController` + `OpenAIWhisperSTTProvider`（按住说话/空格）
- 展示页 + 右侧衣橱抽屉：`WardrobeShowroomUI` / `WardrobeUIController`

---

## 1) 体验目标（用户无需配置也“像女友”）

### 1.1 初次进入
- 点 Play 直接进入“展示页”（已做）：背景氛围 + 角色居中偏右 + 入口按钮。
- 无需打字：默认出现“按住说话”按钮与空格 PTT。
- 首次运行自动生成“人设档案”与“关系状态”默认值。

### 1.2 对话体验
- 回复更短、更口语、更有情绪与语气词（撒娇/害羞/小抱怨/夸夸）。
- 语音输出的语速/音高/停顿随 emotion 变化（“像真人在说话”）。
- 具备“主动开口”：空闲一段时间会主动搭话（可配置频率、可关闭）。

---

## 2) 人设系统（Persona）

### 2.1 Persona 数据（可持久化）
- `petName`：默认“小优”
- `userNickname`：默认“你”
- `relationshipLevel`：初始 1
- `relationshipXp`：初始 0
- `personaStyle`：默认“温柔甜系/有点傲娇/爱打扮”
- `hardBoundaries`：默认边界提示模板（见 2.3）

### 2.2 系统提示（System Prompt）策略
- 固定“底座 prompt”：定义身份、语气、风格、输出格式（emotion + 文本）。
- 动态注入：
  - 用户昵称、人设、关系等级、近期记忆摘要（见 3.3）
  - 今日日期、闹钟/事件上下文（如有）

### 2.3 陪伴安全边界（默认开启，可在设置中关闭）
- 避免强依赖措辞：不承诺“只属于你/你不能离开我”等。
- 遇到自伤/极端内容时：引导联系现实帮助资源（保持简短、不说教）。

---

## 3) 长期记忆（Memory）

### 3.1 记忆分层
- **短期对话历史**：保留最近 N 条（已存在 `maxHistoryMessages`）
- **长期摘要（Summary Memory）**：每当短期消息超阈值时做一次“总结归档”
- **偏好/事实（Facts）**：从对话中抽取“喜欢什么/讨厌什么/常用称呼/重要日期”

### 3.2 记忆存储（持久化字段）
- `longTermSummary`：一段可读摘要（200~600 字）
- `factsJson`：简单键值 JSON（如 `{ "喜欢的风格":"粉紫金", "讨厌":"太啰嗦" }`）
- `lastProactiveAt`：上次主动开口时间戳

### 3.3 记忆写入策略
- 每次 AI 回复后，若累计消息超过阈值：
  - 触发一次“摘要任务”：把最近一段对话压缩为摘要，更新 `longTermSummary`
  - 从对话中抽取 1~3 个 facts（轻量）
- 约束：摘要与抽取都走同一个 LLM（先做最小可用，不引入向量库）

---

## 4) 主动陪伴（Proactive Companion）

### 4.1 触发条件（默认）
- 空闲 N 秒无输入（例如 120~300 秒），并且窗口未被“点击穿透隐藏”
- 每日首次打开问候
- 闹钟触发（已有 AlarmManager）

### 4.2 内容策略
- 优先围绕“穿搭/试衣间”话题：夸穿搭、建议换色、提醒 SSR、新增搭配挑战
- 次级：轻日常（天气/心情/提醒喝水），避免打扰（频率可调）

---

## 5) 情绪语音（Emotion-aware TTS）

### 5.1 情绪映射
沿用 LLM 回复前缀解析 emotion（已存在），扩展一组映射：
- `happy`：语速略快，音高略高
- `shy`：语速略慢，音量略小，更多停顿
- `angry`：语速稍快，音高稍低，少停顿
- `sad`：语速慢，音高略低
- `idle/neutral`：默认

### 5.2 Azure SSML 输出（最小可用）
- 在 `AzureTTSProvider` 中根据 emotion 包一层 `<prosody rate='' pitch='' volume=''>`
- 仍保持现有 voiceName 不变；后续可加 `voiceName` 轮换与风格参数（不强依赖 Azure 的扩展标签）

---

## 6) UI（更像暖暖的“女友桌面”）

### 6.1 展示页叠加层（Showroom Overlay）
- 增加“关系状态小角标”：Lv、称呼、当日心情 icon（极简）
- 增加“对话气泡”区域：AI 回复以气泡呈现（同时保留 chat history 面板作为开发/调试模式）

### 6.2 右侧抽屉（Wardrobe Drawer）
- 继续在现有主题上做精修：
  - 按钮 hover/press 动画节奏
  - 卡片阴影/分层/字体层级
  - SSR 角标与扫光更“金”

---

## 7) 验收标准（MVP）
- 点 Play：展示页可见，角色可拖拽旋转/缩放，右侧抽屉可开合。
- 空格按住说话：能识别中文 → AI 回复 → 语音播报（有口型）。
- AI 回复稳定带 emotion，语音输出随 emotion 有明显差异。
- 关闭/开启主动开口开关后行为符合预期。
- 退出再进：人设、记忆摘要、偏好信息仍存在（持久化生效）。
