# 桌宠 UI 重构：右键菜单 + 轻量浮层（Design）

## 目标

把当前“移动端大面板”交互改为更桌宠的“隐形 UI”：

- 默认屏幕上只有角色 + 气泡 + toast
- 功能入口以 **右键菜单（只点到角色才弹出）** 为主
- 衣橱 / 拍照 / 设置改为 **轻量浮层/抽屉**，用完即退场

## 非目标

- 不在本阶段做“暖暖级单件衣服精细穿搭”（本阶段仍以套装库/整套切换为主）
- 不在本阶段做新美术/新 UI 皮肤资源（保持现有玻璃质感主题）

## 交互设计

### 1) 右键菜单（核心入口）

**触发规则**

- 鼠标右键时，从主相机对鼠标位置做射线检测（Raycast）
- 仅当命中“当前角色”的 Collider 时显示菜单
- 点击空白不弹出（避免像系统菜单）

**Collider 兜底**

- 若当前角色实例上没有任何 Collider（VRM 常见），运行时自动补一个 `CapsuleCollider`
- Collider 尺寸基于 Renderer bounds 估算

**菜单项**

- 聊天：聚焦输入/打开聊天小条
- 拍照：打开/关闭拍照浮层
- 衣橱：打开/关闭衣橱抽屉
- 套装：扫描 `Mods/character_*` 的下拉列表，选择即切换
- 设置：打开/关闭设置浮层
- 退出：可选（或只放在设置里）

**关闭规则**

- 选中任意菜单项后自动关闭
- Esc 关闭
- 点击空白关闭

### 2) 衣橱：抽屉为主，高级筛选收纳

- 保留右侧抽屉作为主体
- 左侧 Filter 面板默认不常驻
- 常用项常驻：搜索 + 分类 Tabs
- 高级项（稀有度/标签/排序/收藏/仅拥有）收纳进“筛选”按钮的弹出浮层

### 3) 拍照：紧凑控制条

- 保留现有能力：镜头/构图/滤镜/滑杆/辅助线/预设/导出开关/全屏预览
- UI 以紧凑布局呈现，默认折叠高级项
- Esc 一键退出

### 4) 设置：小浮层 + 密钥安全

- 保留 Key/BaseUrl/Model 三项
- Key 输入支持“隐藏/显示”切换（默认隐藏）
- Key 永不写入仓库，只写入本机存档

## 技术方案（实现要点）

### 1) 输入与右键命中

- 新增 `PetContextMenuController`：
  - Update 监听右键
  - Raycast 命中判定与“当前角色 root”绑定
  - 动态生成 UGUI 菜单（复用 DefaultControls + WardrobeThemeFactory）

### 2) 角色可点击性保障

- 在 `CharacterModLoader.BindCharacterToSystems` 后调用 `EnsureClickableCollider(characterObj)`
- 对 `Renderer.bounds` 做聚合，生成 CapsuleCollider（center/height/radius）

### 3) UI 组件重用

- 衣橱/拍照/设置继续使用现有脚本，但默认隐藏，改为由菜单触发显示
- 将“高级筛选”抽为可隐藏的 panel

## 验收标准

- 右键必须“点到角色才弹出”
- 默认无大面板常驻
- Esc/点空白能快速收起菜单与浮层
- VRM 套装切换能在右键菜单内完成
