# 下一步设计：衣橱切换更丝滑（无闪烁）· 拍照模式产品化（背景/灯光/姿势）

**Goal:** 在已有“整屏淡出→入场节奏”的基础上，进一步消除切换时的空白闪烁与布局跳动；同时把拍照模式做成用户可用的“摄影棚出片”：背景、灯光 preset、姿势一键、截图一键保存。

**约束**
- 不引入侵权素材；背景/光效均程序化或纯色/渐变。
- 不引入新渲染管线；保持 Built-in + UGUI。

---

## 1) 衣橱切换更丝滑（No Flicker）

### 1.1 Loading Overlay（极短）
- 切换分类/筛选刷新时，展示一层轻薄玻璃 Loading（0.12~0.18s），避免“空白瞬间”。
- overlay 只遮盖列表区域（不遮按钮），并自动淡入淡出。

### 1.2 淡出含轻位移
- 旧卡片淡出同时轻下移 2px，提升“过渡感”。

### 1.3 入场基准点修正
- 目前卡片入场基于 cached basePos，若布局重排可能导致偏移。
- 修正为：每次入场都以当前 layout 计算出的 anchoredPosition 作为 basePos。

---

## 2) 拍照模式（Photo Mode v2）

### 2.1 UI 入口
- 展示页新增按钮：`拍照`
- 点击弹出面板（玻璃拟态）：
  - 背景：透明 / 纯色（粉/蓝/奶油）/ 柔和渐变
  - 灯光：棚拍·暖 / 棚拍·冷 / 粉紫金
  - 姿势：idle/happy/shy/cute（触发 Animator Trigger）
  - 按钮：`保存照片`

### 2.2 截图规则
- 调用现有 `PhotoModeManager.TakeScreenshot`
- 自动填充 `uiElementsToHide`：拍照面板、衣橱按钮、设置按钮、麦克风按钮、气泡等（避免 UI 入镜）
- 可选：保留背景（用户选择透明背景时背景 UI 也隐藏）

---

## 3) 验收标准
- 切换筛选/分类时没有明显空白闪烁，过渡更稳定
- 拍照面板可切背景/灯光/姿势并保存截图到 `Screenshots/`
