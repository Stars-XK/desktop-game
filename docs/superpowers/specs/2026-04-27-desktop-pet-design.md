# 3D AI 桌宠技术方案与需求分析

## 1. 项目概述
本项目旨在开发一款高自由度、支持自定义模型与复杂换装的 3D AI 桌面宠物。项目以“闪耀暖暖”级别的换装系统为标杆，结合大语言模型（LLM）、语音合成（TTS）以及AI动作生成技术，提供丰富的桌面互动与拍照体验。游戏去除传统的数值养成与经验机制，专注于纯粹的换装搭配与AI陪伴。

## 2. 核心架构与技术栈
- **基础引擎**: Unity 3D
- **桌面级特性**: 
  - 支持 Windows 无边框透明窗口（通过调用 Windows API 实现 DwmExtendFrameIntoClientArea 等）。
  - 鼠标穿透与命中测试：仅在点击到角色模型时拦截鼠标事件，其余区域鼠标穿透。
- **资源管理**: Addressables / AssetBundle 系统。用于加载外部导入的模型、服装、材质包。
- **持久化存储**: JSON / SQLite 用于保存玩家的搭配方案、设置以及聊天历史。

## 3. 需求分析与核心系统设计

### 3.1 复杂换装与模型系统 (核心)
- **底层逻辑**: 骨骼复用与部件拆分。
- **服装部件化**: 将头发、上衣、下装、鞋子、饰品等拆分为独立的 Mesh，绑定到相同的骨骼架构上。
- **动态骨骼/物理引擎**: 接入 Magica Cloth 2 或 Dynamic Bone，实现头发、裙摆的自然物理飘动，并在打包 AssetBundle 时配置好物理参数。
- **材质渲染**: 使用高质量的卡通渲染（如 UTS2 或自定义 Shader），支持复杂的材质表现（蕾丝、金属、流光）。
- **模组(Mod)导入**: 提供一个 Unity SDK 工具包，创作者可以在 Unity Editor 中将自定义的模型、服装打成 `.bundle` 文件，放入游戏指定目录后，游戏在运行时动态加载并解析。

### 3.2 AI 交互与 TTS 语音系统
- **灵活的接口架构**: 采用 Provider 模式设计，支持玩家在设置中自由切换服务商。
- **LLM 大语言模型**:
  - **云端**: 接入 OpenAI、Claude、DeepSeek 等标准 API。
  - **本地**: 接入 Ollama，支持离线运行，保护隐私。
  - **Prompt 工程**: 赋予桌宠特定的性格设定，并要求 LLM 在输出文本的同时，附带 JSON 格式的情感标签（如 `[emotion: happy]`）。
- **TTS 语音合成**:
  - **云端**: 接入 Azure TTS、阿里等提供自然拟真声音的服务。
  - **本地**: 接入 VITS 或 Edge TTS 本地/免费方案。支持口型同步（LipSync），根据音频响度或音素动态驱动角色的嘴部 BlendShape。

### 3.3 动作生成与互动系统
- **混合动作模式**:
  - **预制高质量动作**: 根据 LLM 输出的情感标签，触发对应的预制精美动作（如开心时的转圈、打招呼），保证视觉上的“闪耀暖暖”级别美感。
  - **AI 动作生成**: 接入文本生成3D动作的 API（如 MotionGPT 或类似开源模型），对于特定指令（如“跳一段随机的舞”），实时下载并重定向(Retargeting)骨骼动画到角色身上。
- **桌面互动**:
  - **鼠标交互**: 抚摸（触发特定动画和物理反馈）、拖拽移动位置。
  - **待机行为**: 随机触发的待机动画、打瞌睡、看向鼠标指针（IK头部追踪）。

### 3.4 拍照与搭配系统
- **全解锁衣橱**: 抛弃经验值和关卡解锁，所有导入或内置的服装默认全部可用。
- **搭配存档**: 玩家可以将当前的一套外观（发型、五官、服装）保存为“预设套装”，一键切换。
- **专业拍照模式**:
  - **环境控制**: 提供纯色背景、透明背景或虚拟3D影棚背景。
  - **灯光系统**: 允许玩家调整主光源、轮廓光、补光的位置与颜色。
  - **自由镜头与动作控制**: 在拍照模式下可暂停动画，微调角色的关节（FK），或者选择特定的Pose，调整 FOV 后输出高清透明 PNG 图片。

## 4. 后续开发计划 (Roadmap)
1. **Phase 1: 基础框架与透明窗口** (搭建 Unity 工程，实现透明背景与鼠标穿透)。
2. **Phase 2: 换装系统核心** (实现骨骼重组逻辑，制作 SDK 工具包，跑通一个导入 Demo)。
3. **Phase 3: AI 大脑与语音** (接入 LLM API 与 TTS，实现基础聊天与口型同步)。
4. **Phase 4: 动作与互动** (接入 IK、动画状态机，实现情感驱动动作与 AI 动作生成)。
5. **Phase 5: 拍照与打磨** (实现 UI 系统、拍照模式、灯光控制与最终优化)。
