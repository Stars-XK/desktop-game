# 下一阶段（产品化）设计规范：暖暖级观感 · 女友互动 · 大模型兼容

**Goal:** 在现有“展示页 + 右侧衣橱抽屉 + 语音输入/输出 + 人设/记忆/主动陪伴”的基础上，完成产品化三件事：
1) 视觉观感更高级（不依赖侵权素材）
2) 互动更像“女友”而不是问答机器人
3) LLM 接入变成“OpenAI Compatible 统一配置”，可切换市面兼容模型

**资源策略（用户已同意）**
- 默认：仅引入“可商用免费、可再分发”资源，并将 LICENSE 一并放入项目。
- 可选：列出付费资源清单，用户购买后再接入。
- 在资产缺位时：先把镜头/灯光/后期/动效/UI 质感拉满。

---

## 1) 视觉与质感（不靠原版资产）

### 1.1 Showroom 试衣间氛围（代码可落地）
- 三点布光：主光/补光/轮廓光（软阴影、柔光）
- 背景分层：渐变 + 轻粒子/漂浮光点（程序化或自制贴图）
- 镜头：全身/半身/近景切换（轻缓动），默认全身；拖拽旋转/滚轮缩放保留

### 1.2 UI 观感升级（代码可落地）
- 展示页增加“气泡对话”层：AI 回复直接出现在暖暖风气泡中（替代开发式 chat history）
- 保留“开发/调试聊天面板”但默认隐藏（避免破坏沉浸）

---

## 2) 互动更像女友（状态机 + 反馈库）

### 2.1 触摸/互动入口
- 头部/身体/拖拽：不同交互触发不同“反应模板”
- 每次互动会提升关系 XP，达到阈值自动升级 Lv（并触发一句“升级反馈语音/文本”）

### 2.2 反应库（最小可用）
- 互动触发不直接写死台词：用“短 prompt 模板”让 LLM 生成更自然的反应
- 内置冷却，避免连续刷屏

---

## 3) OpenAI Compatible 大模型接入（统一配置）

### 3.1 配置项
- `baseUrl`（例如 `https://api.openai.com` 或第三方兼容端点）
- `apiKey`
- `model`
- 兼容路径：默认使用 `/v1/chat/completions`

### 3.2 UI 与存档
- 设置面板新增：Base URL、Model 输入框
- 存档新增：`llmBaseUrl`、`llmModelName`（默认 OpenAI）

---

## 4) 资产清单输出（交付物）

### 4.1 免费可商用资源清单（带许可）
- 人物：可商用角色（带表情/骨骼优先）
- 动作：待机/撒娇/害羞/生气/夸夸
- UI 字体：可商用中文字体（或系统字体方案）
- 粒子/装饰：CC0/可商用贴图包

### 4.2 付费资源建议清单
- Unity Asset Store / Sketchfab 商用许可
- 目标：更接近“暖暖”质感的角色与动画

---

## 5) 验收标准（本阶段）
- 设置面板：可配置 baseUrl + model + key；切换到兼容端点可正常对话
- 展示页默认出现气泡对话；调试聊天面板默认隐藏
- 互动：摸/拖拽能触发不同“女友式”反应，并提升关系等级
- 输出一份资源清单文档（免费可商用 + 付费建议）
